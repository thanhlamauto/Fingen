\begin{abstract}
Training fingerprint classifiers is still bottlenecked by data access: useful
training sets need many impressions per finger, but biometric data is expensive
to collect, difficult to share, and constrained by privacy regulation.  The
problem becomes especially concrete when a lab or company deploys a newly
purchased sensor: only a small calibration set may be available, while
collecting enough labeled impressions to retrain a robust recognizer is slow
and legally sensitive.  We present \method, a lightweight synthetic-data
pipeline for this setting.  Stage 1 samples privacy-preserving virtual
fingerprint identities from an unconditional latent diffusion prior.  Stage 2
renders multiple impressions from each identity under target-sensor appearance
using ridge conditioning, pose-mask alignment, and a small sensor LoRA adapter.
Compared with recent Stable-Diffusion-based fingerprint generators, our active
Stage-2 transfer path uses only 23.39M parameters, reducing the generator
backbone size by 97.3--98.5\%.  This compact design targets low-end GPU
research hardware while preserving explicit ridge control for recognition
utility.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

Fingerprint recognition research is constrained by a specific data problem:
useful training sets need both many identities and repeated impressions of each
identity, but biometric data is expensive to collect, difficult to share, and
privacy-sensitive.  Each image contains two kinds of information that must be
modeled separately.  The ridge flow, ridge frequency, and minutiae layout define
the finger identity; sensor contrast, background texture, pressure, crop, pose,
and local artifacts define the acquisition domain.  A recognizer needs both
axes of variation, but public datasets rarely provide them at scale.

We study the target-sensor augmentation setting.  A lab may have a small
calibration set from a newly deployed sensor, but not enough users or repeated
impressions to retrain a robust classifier.  The goal is to synthesize extra
training impressions that (i) cover many virtual finger identities, (ii) match
the target sensor's foreground mask, tone, contrast, and artifact statistics,
and (iii) preserve identity-defining ridge/minutiae structure across multiple
impressions of the same virtual finger.  This is harder than generic image
generation: a visually plausible fingerprint is not necessarily useful if ridge
breaks, ridge merges, or displaced minutiae change the identity evidence seen by
a matcher.

This target-sensor problem also differs from ordinary data augmentation.  Simple
geometric or photometric transforms can perturb an existing real impression, but
they cannot create new identities or reproduce the full distribution shift
introduced by a different acquisition device.  Conversely, a generator trained
for one sensor can produce many samples, but those samples may not transfer when
the deployment sensor has a different pose footprint, ridge/valley contrast, or
background pattern.  The practical requirement is therefore a generator that can
separate identity from sensor rendering and adapt the rendering component from a
small target-sensor set.

The same pressure has already made synthetic data an important direction in
face recognition, where rendered and diffusion-generated identities are used to
reduce dependence on web-crawled personal images and consent-limited
collections~\cite{bae2023digiface,kim2023dcface}.  Fingerprints pose a related
but more structural version of the problem: useful synthetic data must preserve
ridge and minutiae identity while producing realistic impression-level changes
under different acquisition devices.

Recent fingerprint generators address this gap by synthesizing multiple
impressions of each synthetic finger.  FPGAN-Control~\cite{shoshan2023fpgan}
is the closest baseline for our setting because it is trained and evaluated on
NIST SD302/N2N and explicitly controls fingerprint appearance.  Its practical
adaptation path is less direct, however: a newly introduced sensor generally
requires retraining the full generator rather than attaching a small
sensor-specific adapter.  The released setting is also tightly coupled to the
SD302/N2N data organization and sensor taxonomy, which makes the framework
harder to port when the target dataset has different acquisition protocols or
sensor styles.  GenPrint~\cite{grosz2024genprint} and Ridge Scale Aligned
Diffusion~\cite{hsu2026ridge} extend the direction with Stable-Diffusion-based
pipelines, ControlNet-style structure guidance, and image/text style
conditions.  These designs are powerful, but the inherited backbone is large:
the public Stable Diffusion v1 configuration uses an 860M-parameter UNet, and
ControlNet-style inference adds a second large conditional branch.  That level
of compute is not always available to smaller biometrics labs that want to
prototype recognition models on low-end GPUs.

This work explores a more pragmatic point in the design space.  Rather than
fine-tuning a large general-purpose text-to-image model, \method uses a compact
fingerprint-specific latent diffusion model.  Identity generation is deliberately
kept unconditional so that the first stage learns a fingerprint identity prior
without binding sampled ridges to a reference sensor style.  Sensor appearance
is handled after ridge extraction through the Stage-2 renderer and its sensor
adapters.  Identity preservation is handled through an explicit ridge condition
in the second stage.  Finally, sensor adaptation is performed with Stage-2 UNet
LoRA only: the first-stage identity generator and the structural control path
remain frozen.

Our contributions are:
\begin{itemize}
    \item A lightweight two-stage latent diffusion pipeline that separates
    unconditional fingerprint identity generation from ridge-conditioned,
    sensor-specific impression rendering.
    \item A Stage-2 UNet LoRA adaptation framework for transferring the
    generator to new sensors while keeping the identity prior and ridge-control
    path frozen.
    \item Recognition, matching, and style-clustering diagnostics on SD302A and
    FVC 2004 showing when synthetic data helps and where low-contrast sensors
    remain challenging.
\end{itemize}

\section{Related Work}
\label{sec:related}

\paragraph{Synthetic fingerprints.}
Classical systems such as SFinGe~\cite{cappelli2002synthetic} construct
fingerprints using hand-designed ridge flow, shape, and noise models.  Recent
GAN-based methods improve realism and scale.  PrintsGAN~\cite{engelsma2022prints}
generates large sets of synthetic fingers with multiple impressions per finger,
which is essential for training embedding-based recognizers.  FPGAN-Control
\cite{shoshan2023fpgan} further disentangles identity and appearance factors,
allowing image appearance attributes such as sensor and pressure to be varied
while keeping identity fixed.  Because FPGAN-Control is evaluated on the NIST
SD302/N2N setting, we use it as our main competitor.

\paragraph{Fingerprint diffusion.}
Latent diffusion models reduce pixel-space diffusion cost by denoising in a
compressed latent representation~\cite{rombach2022ldm}.  GenPrint
\cite{grosz2024genprint} applies a multimodal latent diffusion model to
fingerprints, using text prompts and image style embeddings to control
acquisition type, sensor, quality, and identity.  GenPrint is a strong
reference point for universality, but it is built on a larger Stable
Diffusion-style backbone and a broad multi-dataset training recipe.  Our goal
is complementary: provide a smaller, faster pipeline that can be trained,
sampled, and adapted by recognition teams with modest hardware.

\paragraph{Structural control and efficient adaptation.}
ControlNet~\cite{zhang2023controlnet} adds trainable control branches to a
frozen diffusion model for spatially aligned conditions.  For fingerprints,
ridge maps are a natural structural condition because they encode the ridge
flow and minutiae layout used by matchers.  LoRA~\cite{hu2022lora} provides a
low-rank adaptation mechanism for fine-tuning large neural networks with few
trainable parameters.  We use LoRA as a practical sensor-adaptation mechanism
by tuning only Stage-2 UNet adapter weights.

\section{Method}
\label{sec:method}

\subsection{Preliminaries: Latent Diffusion Models}

Following DDPMs~\cite{ho2020ddpm} and latent diffusion
models~\cite{rombach2022ldm}, we formulate generation as iterative denoising
in a compressed fingerprint latent space.  Given an image latent $z_0$, the
forward process adds Gaussian noise over $T$ timesteps:
\[
    q(z_t \mid z_0) =
    \mathcal{N}\left(
        \sqrt{\bar{\alpha}_t} z_0,
        (1-\bar{\alpha}_t)\mathbf{I}
    \right),
\]
or equivalently
$z_t=\sqrt{\bar{\alpha}_t}z_0+\sqrt{1-\bar{\alpha}_t}\epsilon$ with
$\epsilon \sim \mathcal{N}(0,\mathbf{I})$.  A denoising UNet is then trained to
predict the sampled noise:
\[
    \mathcal{L}_{\text{ldm}} =
    \mathbb{E}_{z_0,t,\epsilon}
    \left[\left\|
        \epsilon - \epsilon_\theta(z_t,t,c)
    \right\|_1\right],
\]
where $c$ denotes an optional condition.  In our pipeline, $c=\varnothing$ for
the Stage-1 identity prior and $c=r$ for the Stage-2 ridge-conditioned
renderer.  At inference time, we start from Gaussian noise and use DDIM-style
sampling~\cite{song2020ddim} to recover a clean latent in fewer reverse steps.
\subsection{Overview}

\method is a two-stage generator.  Stage 1 samples a synthetic fingerprint
identity from an unconditional fingerprint latent prior.  Stage 2 renders
multiple impressions of that identity under target sensor appearances.  Both
stages operate on $512 \times 512$ grayscale images encoded into
$128 \times 128 \times 3$ VQ-VAE latents.  The VQ-VAE is frozen.

Let $x$ be a fingerprint image and $z=E(x)$ its latent.  Stage 1 models
$p_\theta(z)$ and generates a base identity image $\hat{x}^{id}$.  A ridge
extractor produces a control map $r = R(\hat{x}^{id})$.  Stage 2 then samples
impressions $\hat{x}^{imp}_1,\ldots,\hat{x}^{imp}_K$ conditioned on the ridge
map and the selected target-sensor rendering parameters.

\subsection{Fingerprint VQ-VAE Latent Space}

We build the generator on a hierarchical fingerprint representation that
combines discrete latent modeling with latent diffusion using a VQ-VAE
codebook latent~\cite{oord2017vqvae,pan2026impose}.  The goal of this first
representation stage is to compress away redundant grayscale variation while
keeping the ridge topology that defines the identity of a finger.  For an
aligned $512 \times 512$ fingerprint image $x$, the encoder $E$ first produces
a continuous latent grid
\[
    z_\ell = E(x) \in \mathbb{R}^{128 \times 128 \times 3}.
\]
Each local latent vector is then replaced by the nearest entry in a learned
codebook $\mathcal{C}=\{e_k\}_{k=1}^{N_C}$:
\[
    k^\star(i,j)=
    \arg\min_{k \in \{1,\ldots,N_C\}}
    \left\|z_\ell(i,j)-e_k\right\|_2,
    \qquad
    z_q(i,j)=e_{k^\star(i,j)}.
\]
In our implementation, the codebook contains $N_C=4096$ vectors and each vector
has dimension 3, matching the three latent channels.  The decoder $D$ maps the
quantized latent $z_q$ back to image space.  Because similar ridge fragments
share codebook entries, this discrete representation reduces pixel-level
redundancy while still modeling the joint distribution of local ridge
orientation, ridge frequency, and minutiae-like structures.

The spatial bottleneck is intentionally conservative.  The $128 \times 128$
latent grid corresponds to only $4\times$ spatial downsampling from the input
image.  This is larger than the usual latent grid in many natural-image latent
diffusion systems, but it is necessary for fingerprints: once ridge flow is
blurred or minutiae are merged inside the autoencoder, the downstream diffusion
model cannot reliably recover the original identity.  After training this
fingerprint VQ-VAE, we freeze the encoder, codebook, and decoder.  Stage 1
learns a diffusion prior over $z_q$, and Stage 2 reuses the same decoder-facing
latent space to render sensor-specific impressions from ridge conditions.

The first diffusion stage establishes the synthetic identities used by the
entire pipeline.  It is trained on the codebook-quantized fingerprint latents
rather than on raw pixels.  Given a training image, we encode and quantize it
into $z_q$, sample a diffusion timestep $t$, and corrupt the latent with
Gaussian noise to obtain $z_t$.  A U-Net denoising network then predicts the
noise in the VQ-VAE latent space:
\[
    \mathcal{L}_{\text{stage1}} =
    \mathbb{E}_{z_q,t,\epsilon}
    \left[\left\|
        \epsilon - \epsilon_\theta(z_t,t)
    \right\|_1\right].
\]
The model is deliberately unconditional: it receives no sensor label, reference
image set, text prompt, or learned style descriptor.  This keeps the first
stage focused on sampling new ridge identities instead of entangling identity
with a particular sensor appearance.

At inference time, we follow the IMPOSE-style notation shown in
Figure~\ref{fig:stage1_identity_generation}.  We draw random latent noise
$z_T \sim \mathcal{N}(0,\mathbf{I})$ and iteratively apply the U-Net denoiser
for $T$ reverse steps; $z_{T-1}$ denotes an intermediate denoised latent in this
reverse chain, and the ``$\times T$'' mark in the figure indicates this
repeated denoising process.  The resulting continuous estimate $z_0$ is not
decoded directly.  Instead, we pass it through the same codebook
nearest-neighbor projection used by the VQ-VAE:
\[
    z'_0(i,j)=
    e_{\arg\min_k \left\|z_0(i,j)-e_k\right\|_2}.
\]
This final quantization step forces the generated latent back onto the learned
fingerprint manifold and prevents the decoder from receiving arbitrary
off-codebook features.  We denote the frozen identity decoder as $D_I$, and it
produces the generated rolled identity image $I_r=D_I(z'_0)$.  In the rest of
this paper, this image is the same base identity image denoted
$\hat{x}^{id}$.  It is not treated as the final synthetic sample; it is the
identity reference from which we extract the ridge condition for Stage 2.
Sensor style, pose variation, and local acquisition artifacts are introduced
only in the second stage.

\subsection{Identity-Consistent Control via Sauvola Binarization}

The central challenge in cross-sensor fingerprint generation is to change
sensor style while preserving identity.  Fingerprint recognition has little
tolerance for ridge and minutiae deviations: a small ridge break, ridge merge,
or displaced minutia can shift the perceived identity even when the global
finger shape still looks plausible.

We found that common identity-consistency signals are not reliable enough for
this setting.  Fixed-length descriptor supervision, learned ridge-extractor
losses, and minutiae-based rigid alignment can all be brittle when the source
identity image and target-sensor samples differ in acquisition time, contact
area, pressure distortion, and local sensor artifacts.  Instead, we use a
direct control view: if the generated impression follows the same ridge-level
pixel structure as the Stage-1 identity image, then identity preservation is
enforced before any recognizer loss is needed.

We use a high-contrast ridge map as the identity injection signal.  Binary
ridge images are useful for two reasons.  First, their contrast exposes the
identity-bearing ridge and valley pattern clearly.  Second, because the map is
extracted from the Stage-1 identity image itself, it provides a natural
ridge-level correspondence target for every target sensor rendering.

Learned ridge extractors can fail on low-contrast or artifact-heavy regions and
may hallucinate ridge evidence where the image is ambiguous.  To keep the
control signal deterministic, we adopt Sauvola local adaptive
binarization~\cite{sauvola2000adaptive}.  For a normalized, contrast-inverted
grayscale fingerprint $I'(x,y)$, the local mean $m(x,y)$ and standard
deviation $s(x,y)$ are computed in a $w \times w$ neighborhood:
\begin{equation}
\begin{aligned}
    m(x,y)
    &= \frac{1}{w^2}
    \sum_{i=x-\rho}^{x+\rho}
    \sum_{j=y-\rho}^{y+\rho}
    I'(i,j), \\
    s(x,y)
    &=
    \sqrt{
    \frac{1}{w^2}
    \sum_{i=x-\rho}^{x+\rho}
    \sum_{j=y-\rho}^{y+\rho}
    \left[I'(i,j)-m(x,y)\right]^2
    },
\end{aligned}
\end{equation}
where $\rho=(w-1)/2$ and we set $w=11$ to match the typical ridge width in our
$512 \times 512$ fingerprints.  The adaptive threshold is
\begin{equation}
    T(x,y)=m(x,y)
    \left[
        1+k\left(\frac{s(x,y)}{R}-1\right)
    \right],
\end{equation}
with dynamic range constant $R=128$ and correction parameter $k=0.007$.  This
threshold raises discrimination in textured ridge regions and suppresses
background responses in low-variance regions.

Given a foreground mask $\mathcal{M}$, the binary ridge-control image is
\begin{equation}
    B(x,y)=
    \begin{cases}
        0, & \text{if } (x,y)\in\mathcal{M}
        \text{ and } I'(x,y)>T(x,y),\\
        255, & \text{otherwise}.
    \end{cases}
\end{equation}
We then apply morphological opening with a $2 \times 2$ structuring element to
remove isolated noise.  Figure~\ref{fig:sd302a_sauvola_examples} shows real
SD302A samples and the corresponding ridge maps extracted with these settings.
Sauvola ridges are sometimes less smooth than the outputs of specialized ridge
extractors, but the deterministic procedure stays faithful to the visible
texture and avoids learned hallucination in sparse or damaged regions.  We use
the resulting binary map as the ridge condition $r$ for Stage 2.

\subsection{Ridge-Conditioned Instance Generation}

Stage 2 receives the Sauvola ridge image $r$ and target sensor rendering
information.  The ridge map is encoded by a ControlNet-style branch initialized
from the Stage-1 UNet where compatible.  The denoising UNet predicts the
diffusion noise using the noisy latent, timestep, and residual control
features:
\[
    \epsilon_\phi(z_t,t,r) =
    U_\phi\left(z_t,t,C_\phi(z_t,t,r)\right).
\]
The base Stage-2 training loss is again an L1 noise-prediction objective.
During controlled synthesis, the same identity ridge can be paired with
multiple target-sensor adapters or rendering settings to create cross-sensor
impressions.

\subsection{Stage-2 UNet LoRA for Sensor Adaptation}

For a new sensor, we use the available target-sensor samples only to adapt
appearance.  The VQ-VAE, Stage-1 identity model, ridge extractor, and
ControlNet structural path remain frozen.  We insert LoRA modules into the
Stage-2 denoising UNet middle self-attention $qkv$ and output projection
layers, then optimize only these LoRA weights.
The intent is to change sensor texture, contrast, background statistics, and
local artifact style while preserving the behavior of the ridge-control path.
At inference time, the adapter is selected by sensor, while the input ridge and
sampling settings provide identity and appearance control.

\subsection{Pose-Mask Alignment and Control Perturbation}

A direct ridge condition can over-constrain the output pose: all target sensor
instances may inherit the same fingerprint placement as the Stage-1 identity
image.  To avoid this, we construct a per-sensor pose-mask library from real
target-sensor foreground masks.  For each generated impression, we cycle
through the target sensor's pose masks, align the ridge foreground to the mask
using a bounded similarity transform, and keep only the intersection between
the transformed ridge mask and the target pose mask.  We also apply mild
control perturbations: small rotations, translations, rare horizontal flips,
ridge dropout, speckle, and light morphology.  These perturbations provide
intra-class variation while keeping minutiae topology mostly intact.

\section{Experiments and Results}
\label{sec:results}

\paragraph{Experimental setting.}

NIST SD302A~\cite{nist302} is the main benchmark and the only dataset on which
we compare with FPGAN-Control.  We split real data by finger identity after
stripping sensor and capture suffixes from filenames, and use a held-out
SD302A test split with 400 finger identities and 2,734 images.  Unless stated
otherwise, synthetic training sets contain 2,000 identities with 10 impressions
per identity at $512 \times 512$ resolution.  Stage 1 samples identities from
the unconditional latent prior; Stage 2 renders ridge-conditioned impressions
with DDIM-50 sampling across target sensors.  Recognition utility is evaluated
by training recognizers on real, synthetic, or real-plus-synthetic data and
reporting TAR@FAR=$0.1\%$ on the held-out SD302A test split.  SD301A~\cite{nist301},
SD302B, and FVC 2004~\cite{fvc2004} are used only for additional sensor
coverage and cross-sensor transfer studies.

\subsection{First-Stage Latent Resolution}

instead of a conventional $32 \times 32$ LDM-style latent because fingerprint

spatial resolution, we keep the same trained decoder and compare normal
decoding against a $32 \times 32$ bottleneck proxy obtained by downsampling the
encoded latent and upsampling it back before decoding.  Figure~\ref{fig:latent_resolution_ablation}

$32 \times 32$ proxy removes most ridge detail and turns minutiae-bearing

\subsection{Sauvola Ridge Control vs. Canny Edges}

ControlNet-style image generation often uses Canny edges as a generic spatial
hint, but fingerprints require a different kind of structural signal.  A Canny
map marks intensity transitions, so a single ridge can become two thin contour
boundaries; weak ridges disappear, while crop lines and sensor artifacts can be
promoted to control pixels.  Sauvola binarization instead selects one ridge
phase and produces a dense ridge-support image.  To isolate this conditioning
choice, we compare Sauvola maps with a ControlNet-style Canny preprocessor
before Stage-2 generation.

We sample 160 real SD302A rolled images, 20 per sensor, resize each image to
the $512 \times 512$ condition space, and compute two control maps: our
Sauvola setting and Canny with Gaussian smoothing, non-maximum suppression,
and hysteresis thresholds 100/200.  Table~\ref{tab:sauvola_canny_ablation}
reports foreground-only control-map statistics, while
Figure~\ref{fig:sauvola_canny_ablation} shows representative sensors.  Canny
uses fewer foreground pixels, has no $3\times3$ interior support after edge
thinning, and is roughly $2.8\times$ more fragmented per active pixel.  Its
largest connected support is also much smaller, indicating that the control
branch receives disconnected contours rather than ridge bands.  These results
support using Sauvola as the identity-preserving condition; Canny is effective
for generic object boundaries, but it is a poor phase-preserving ridge signal
for minutiae-sensitive fingerprint synthesis.

\subsection{Stage-2 LoRA Placement}

The adapter location is important because sensor transfer should change
appearance without weakening the ridge-control path.  We therefore place LoRA
on the Stage-2 denoising UNet rather than on the ControlNet branch.  Within
the current compact UNet, the only active self-attention module is the
bottleneck middle block, so the reported experiments fix rank-8 LoRA on the
middle self-attention $qkv$ projection and output projection due to time and
compute-resource constraints.  This gives a
global, low-dimensional appearance handle while keeping the VQ-VAE, Stage-1
identity prior, ControlNet, and base Stage-2 weights frozen.

Table~\ref{tab:lora_placement_ablation} compares natural placement choices.
A $qkv$-only adapter can change attention scores and values but leaves the
attention output remix fixed; a projection-only adapter is cheaper but cannot
change the attention content.  The trained FVC adapters use both parts: across
DB1--DB4 at step 5k, the EMA LoRA update/kernel Frobenius ratio is
75.4--147.1\% for $qkv$ and 86.2--180.3\% for the output projection.  Moving
the same 12.3K LoRA parameters into ControlNet is rejected because it adapts
the ridge-to-residual structural path rather than sensor appearance.  Expanding
LoRA to every Stage-2 self-attention block would require 119.8K trainable
parameters, 9.75$\times$ more than our current adapter, and full Stage-2
fine-tuning would train 23.39M parameters.  This is poorly matched to the
80-image FVC adaptation setting and increases the chance of memorizing sensor
samples rather than learning a reusable style shift.

\subsection{Stage-2 LoRA Rank Capacity}
\label{subsec:lora_rank_capacity}

Placement alone does not determine the adapter capacity.  Even at the same
middle-attention location, the LoRA rank $k$ controls how many independent
sensor-style directions the adapter can learn.  Sensors with clean contrast may
need only a small rank, while sensors with stronger blur, pressure variation, or
background artifacts can need a larger one.  The reported experiments fix rank 8
for all FVC sensors because of time and compute-resource constraints; this should
not be interpreted as claiming that rank 8 is optimal.  The natural deployment
rule is to choose the smallest rank that performs almost as well as the best rank
for that sensor.

For the selected Stage-2 middle-attention adapter, the cost grows linearly:
\begin{equation}
    P(k) = 1536k.
\end{equation}
Rank 1, 2, 4, and 8 therefore use 1.5K, 3.1K, 6.1K, and 12.3K trainable
parameters, respectively.  Under the project time and compute budget, we ran a
reduced retraining sweep for DB1 and DB2 only, using the same reduced synthetic
scale for each candidate rank.  Table~\ref{tab:lora_rank_eer_sweep} reports
only verification EER because the goal of this diagnostic is rank selection, not
a full cross-sensor benchmark.  In this preliminary run, rank 4 gives the lowest
EER for both completed sensors; this should be read as a constrained diagnostic
rather than a claim that rank 4 is universally optimal.

The practical selection rule is deliberately simple.  For each target sensor we
train a small set of candidate ranks, generate the same number of synthetic
prints for each candidate, train the same classifier, and evaluate it on the
real DB\emph{x}\_B split.  We then choose
\begin{equation}
    k_s = \text{smallest rank with }
    \mathrm{EER}_s(k) \le \mathrm{EER}_{s,\mathrm{best}} + \epsilon .
    \label{eq:lora_rank_selection}
\end{equation}
Here $\epsilon$ is a small EER tolerance, e.g. one percentage point.  If two
ranks are effectively tied, the smaller rank is selected.  A rank is rejected
before this step if its generated images fail the ridge/pose quality checks or
show a clear LightGlue structure drop.  In practice,
Table~\ref{tab:lora_rank_eer_sweep} shows how this rule is applied to the
completed DB1/DB2 quick sweep.

\section{Efficiency}
\label{sec:efficiency}

\subsection{Lightweight Design}

The key engineering goal is to make fingerprint synthesis usable by research
groups without large GPU clusters.  Our current Stage-2 TPU training run uses a
compact 64-channel latent UNet and reaches about 7.05 steps/s for global batch
size 256 on TPU v5p-8 after pre-encoding latents, with stable compiled-step
time.  For deployment and sensor adaptation, the important point is that a new
sensor does not require retraining the full generator: only a small Stage-2
UNet LoRA adapter is optimized.  Table~\ref{tab:efficiency} summarizes the
current parameter accounting.  The full two-stage generator, including the
shared VQ-VAE decoder, has 53.58M parameters.  The active Stage-2 transfer path
has 23.39M parameters, which is 97.3\% smaller than the Stable Diffusion v1
UNet alone and about 98.5\% smaller than an SD1.5+ControlNet-scale inference
stack.  For a new sensor, only 12.3K LoRA parameters are trained.
This 12.3K figure corresponds to the rank-8 adapters used in the reported
experiments because of time and compute-resource constraints; the rank-selection rule in
Section~\ref{subsec:lora_rank_capacity} can choose smaller or larger ranks under a
sensor-specific budget.

\section{Limitations}
\label{sec:limitations}

The current system is deliberately lightweight, and this leaves several
limitations.  First, Stage 2 inherits errors from the extracted ridge condition
and target pose mask.  The SD302A-to-DB1 transfer examples show that warping and
clipping can align a source ridge map to a target sensor footprint, but they do
not guarantee that all minutiae-scale evidence survives the pose change.  Better
foreground masks, ridge thinning, and minutiae-aware conditioning would make the
control signal more reliable.

Second, the LoRA adapters are intentionally small: each target sensor trains
only the Stage-2 middle-attention $qkv$ and output-projection adapters.  The
quick DB1/DB2 rank sweep shows that adapter rank can change EER, but the main
recognition experiments still use a fixed rank-8 setting for all FVC databases
due to time and compute-resource constraints.  A complete rank sweep with
retraining at multiple $k$ values is therefore still needed.  The DB2 results
indicate that low-contrast sensors may also need better quality-aware losses,
not only more or fewer LoRA parameters.

Finally, our additional diagnostics should be read as supporting evidence rather
than full biometric validation.  LightGlue measures paired structural agreement
between a binary ridge hint and a rendered gray fingerprint, not a conventional
fingerprint matcher score.  The t-SNE plot uses a fixed image-style descriptor
because the current pipeline has no separate style encoder.  Future evaluations
should include minutiae-level metrics, more real target sensors, and larger
cross-sensor recognition protocols.

\section{Conclusion}
\label{sec:conclusion}

\method{} presents a two-stage fingerprint diffusion pipeline for generating
rolled identities and adapting them to new sensor domains with minimal
trainable parameters.  A compact latent Stage-1 model samples ridge-bearing
identity structure; Stage 2 renders sensor-specific impressions from ridge
conditions, target pose masks, and a small middle-attention LoRA adapter.  This
design separates identity generation from sensor rendering, allowing a new FVC
sensor style to be fitted from only 80 images while keeping the base generator
frozen.

Across SD302A and FVC 2004 experiments, synthetic data is most useful when
combined with real training images: the real-plus-synthetic setting consistently
improves recognition over the real-only baseline.  The LightGlue analysis
supports that Stage 2 preserves much of the paired ridge structure, while the
t-SNE visualization shows that DB1--DB4 LoRA adapters learn distinct synthetic
sensor styles.  The remaining DB2 gap highlights the main open problem:
low-contrast sensors require more quality-aware adaptation than a single fixed
LoRA recipe.  Overall, the results support lightweight ridge-conditioned sensor
adaptation as a practical direction for fingerprint data augmentation, with
future work focused on minutiae preservation, robust pose-mask conditioning, and
hard low-quality sensor transfer.

the second stage renders sensor-specific impressions from the extracted ridge
condition.  For unseen FVC sensors, only Stage-2 UNet LoRA parameters are
fine-tuned.

\subsection{Synthetic Deployment Policies}

For SD302A, we balance target sensors A--H when rendering the final synthetic
set.  In the 500-identity generation run, identity sources were also balanced
across A--H and the 5,000 rendered impressions were distributed nearly evenly
over target sensors.  For FVC 2004, each DB\emph{x} uses a separate LoRA
adapter trained from DB\emph{x}\_A, then renders 500 identities with 10
impressions each for the corresponding DB\emph{x} experiment.
